## git init
git init命令创建一个空的Git仓库或重新初始化一个现有仓库。 

## git clone <url>
克隆项目到本地

## git add <file>
添加文件或文件夹

## git status
git status 命令用于显示版本库和暂存区的状态

## git commit -m "<message>"
为本次提交增加备注

## git push <remote> <branch>
这将把当前分支的更改推送到指定的远程仓库(<remote>)的指定分支(<branch>)

## git pull <remote> <branch>
拉取指定的远程分支 

## git branch
git branch命令用于列出,创建或删除分支

## git branch <name>
在git branch命令的帮助下创建一个新的分支

## git checkout <branch>
切换分支

## git merge <branch>
合并分支到主分支

## git diff
对数据源的不同版本进行比较

## git diff --staged
上述命令将显示已经分档的文件的变化

## git log
上述命令将显示已经分档的文件的变化

## git reset <commit>
以--mixed的方式将当前分支Head重设为<commit>

## git revert <commit>
使用git revert <commit> 可以撤销指定的提交

## git fetch <remote>
用于从远程仓库获取最新的提交历史和分支信息，但并不自动合并或修改本地代码。其中，<remote>是远程仓库的名称，如 origin。

## git rm <file>
只删除工作区的文件

## git stash
git stash命令用于保存当前工作目录的临时状态，包括暂存区和已修改但未暂存的文件。它会将这些修改保存在一个临时区域（即“堆栈”）中，让你能够回到一个干净的工作目录，可以进行其他操作。等到你完成其他任务后，可以再回到之前的状态，继续之前的开发。

## git tag <name>
 git tag <name>就可以打一个新标签
